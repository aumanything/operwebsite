Service AWS Cloud 1. Amazon EC2 (Elastic Compute Cloud)


•รายละเอียด: เป็นบริการที่ให้คุณสร้างและจัดการเซิร์ฟเวอร์เสมือน (Virtual Servers) ที่เรียกว่า “Instances” ซึ่งสามารถปรับขนาดทรัพยากร (เช่น CPU, RAM, Storage) ได้ตามต้องการ • การใช้งาน: เหมาะสำหรับการประมวลผลทั่วไป, รองรับแอปพลิเคชันขนาดใหญ่ และสามารถใช้ทำงานที่ต้องการความสามารถในการปรับขนาดสูง เป็นรูปแบบการสื่อสารระหว่างอุปกรณ์สองชนิดคือไคลเอนต์และเซิร์ฟเวอร์ โดยไคลเอนต์ทำการร้องขอข้อมูลหรือบริการจากเซิร์ฟเวอร์ที่เป็นผู้ให้บริการ เช่น เว็บเซิร์ฟเวอร์หรือฐานข้อมูล เซิร์ฟเวอร์จะตอบสนองต่อคำขอนั้น โมเดลนี้ใช้กันมากในแอปพลิเคชันบนเว็บและบริการออนไลน์ต่าง ๆ


โมเดลการปรับใช้งานการประมวลผลบนระบบคลาวด์ (Cloud Deployment Models) ประกอบไปด้วยรูปแบบหลัก 3 แบบ:
1.Public Cloud: เป็นระบบคลาวด์ที่บริการเปิดให้สาธารณะ โดยผู้ให้บริการ เช่น AWS, Azure ให้บริการทรัพยากรที่โฮสต์ในศูนย์ข้อมูลของพวกเขา
2.Private Cloud: เป็นระบบคลาวด์ที่สงวนไว้สำหรับการใช้งานภายในองค์กรเท่านั้น มีความปลอดภัยสูงและควบคุมได้ง่าย
3.Hybrid Cloud: ผสมผสานระหว่าง Public และ Private Cloud โดยเลือกเก็บข้อมูลที่ต้องการความปลอดภัยสูงใน Private Cloud ส่วนข้อมูลอื่น ๆ จะเก็บใน Public Cloud

Advantages of cloud computing (ประโยชน์ของการประมวลผลบนระบบคลาวด์)
• ความยืดหยุ่นและความสามารถในการปรับขนาด: สามารถปรับขนาดทรัพยากรให้สอดคล้องกับการใช้งานได้ทันทีโดยไม่ต้องลงทุนฮาร์ดแวร์เพิ่มเติม
• ประหยัดค่าใช้จ่าย: ลดต้นทุนด้านโครงสร้างพื้นฐานและการบำรุงรักษา เนื่องจากจ่ายเพียงเท่าที่ใช้จริง
• ความพร้อมใช้งานสูง: ระบบมีการจัดการสำรองข้อมูลและการกู้คืนในกรณีฉุกเฉินที่ช่วยเพิ่มความน่าเชื่อถือ
• ความปลอดภัย: บริการคลาวด์มีมาตรการรักษาความปลอดภัยที่เป็นมาตรฐานและรองรับการปฏิบัติตามกฎหมาย


AWS Global Infrastructure (โครงสร้างพื้นฐานส่วนกลางของ AWS) AWS มีโครงสร้างพื้นฐานแบบกระจายตามภูมิภาค (Regions) และส่วนที่ให้บริการในหลายสถานที่ภายในภูมิภาคนั้นเรียกว่า Availability Zones นอกจากนี้ยังมี Edge Locations สำหรับให้บริการใกล้กับผู้ใช้มากขึ้น ช่วยลดความล่าช้าในการเข้าถึงข้อมูล


Availability Zone (AZ) คือกลุ่มศูนย์ข้อมูลแยกกันหลายแห่งที่ตั้งอยู่ภายในภูมิภาคเดียวกัน (Region) AZ ช่วยให้มีความเสถียรและความปลอดภัยสูง โดยการทำงานร่วมกันในลักษณะสำรองข้อมูลเพื่อป้องกันความผิดพลาดในกรณีที่เกิดปัญหากับศูนย์ข้อมูลใด ๆ


Edge Location คือจุดให้บริการที่ตั้งอยู่ใกล้กับผู้ใช้ ซึ่งช่วยลดระยะเวลาในการเข้าถึงข้อมูลและให้ประสบการณ์ใช้งานที่รวดเร็วขึ้น โดยเฉพาะสำหรับการให้บริการผ่าน Content Delivery Network (CDN) เช่น Amazon CloudFront ที่ให้บริการเนื้อหาและข้อมูลจาก Edge Location ที่ใกล้กับผู้ใช้ที่สุด
